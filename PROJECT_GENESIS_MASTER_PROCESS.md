## Phase 0: Voice Note Intake and Scope Alignment

### Trigger
Stakeholder initiates project with voice note, document, or description

### Tasks
1. Transcribe voice note (if applicable)
2. Extract key elements:
   - Opportunity description
   - Key stakeholders
   - Timeline expectations
   - Budget parameters
   - Success objectives
3. Pre-populate project intake form
4. Generate scope alignment summary
5. Confirm scope with stakeholder

### Deliverables
- [ ] Transcribed voice note
- [ ] Scope alignment confirmation
- [ ] Project intake form completed

### Prompt to Use
> "Please extract the key elements from this opportunity description and create a scope alignment summary. Identify: opportunity, stakeholders, timeline, budget, and objectives."

### SME Team Reference
Reference: MASTER_SME_TEAM_DOCUMENT.md - No experts needed at this phase

## Phase 1: Expert Team Assembly and SWOT Analysis

### Trigger
Scope confirmed by stakeholder

### Tasks
1. Select expert team based on project type
2. Customize team composition for specific needs
3. Identify any new expert bios needed
4. Conduct rapid SWOT analysis
5. Generate initial risk register
6. Assign research tasks to experts

### Expert Team Selection Guide

| Project Type | Core Experts Required |
|--------------|----------------------|
| **Telecom/Tech** | Technology SME, Industry SME (Regional), Financial Analyst, Legal Specialist |

| **Consumer/Retail** | Marketing SME, Operations Expert, Financial Analyst, Consumer Voice |
| **Real Estate** | Real Estate SME, Legal Specialist, Financial Analyst, Regional Expert |
| **Fintech** | Technology SME, Financial Analyst, Legal/Compliance, Fintech Specialist |

### Deliverables
- [ ] Expert team roster with assigned roles
- [ ] SWOT analysis completed
- [ ] Initial risk register created
- [ ] Research tasks assigned

### Prompt to Use
> "Using the Master SME Team Document, please assemble the appropriate expert team for this [project type] opportunity. Conduct a SWOT analysis and create an initial risk register. Reference: MASTER_SME_TEAM_DOCUMENT.md"

## Phase 2: Deep Dive Research and Gap Identification

### Trigger
Expert team assembled, SWOT complete

### Tasks
1. Coordinate parallel expert research
2. Track research task completion
3. Identify emerging gaps and new questions
4. Assign follow-up research tasks
5. Compile findings into integrated analysis
6. Generate gap identification report

### Research Areas

| Area | Key Questions | Expert Assigned |
|------|---------------|-----------------|
| **Market** | Size, growth, segments, trends | Industry SME |
| **Competition** | Players, positioning, threats | Strategy SME |
| **Technology** | Platform, scalability, security | Technology SME |
| **Financial** | Model, projections, unit economics | Financial SME |
| **Legal** | Regulatory, compliance, IP | Legal SME |
| **Operations** | Execution, resources, timeline | Operations SME |
| **Partners** | Reliability, track record, alignment | Due Diligence SME |

### Gap Identification Template

| Gap ID | Description | Impact | Priority | Mitigation Required |
|--------|-------------|--------|----------|---------------------|
| G001 | [Gap description] | Critical/High/Medium/Low | High/Medium/Low | [Yes/No] |

### Deliverables
- [ ] Research task tracker completed
- [ ] Integrated analysis document
- [ ] Gap identification report
- [ ] Follow-up questions documented

### Prompt to Use
> "Please have the assembled expert team conduct deep dive research on [topic/company]. Track all findings and identify gaps. Each expert should provide findings with references. Reference: MASTER_SME_TEAM_DOCUMENT.md"

## Phase 3: Triple Validation and Risk Mitigation

### Trigger
Deep dive research complete

### Tasks
1. Validate all claims through primary SME expert
2. Cross-validate through secondary perspective
3. Triple-check through tertiary perspective
4. Flag any unverified claims
5. Generate validation report with confidence scores
6. Update risk register with research findings
7. Document mitigations for each risk

### Validation Framework

| Claim Category | Confidence Threshold | Validation Method |
|----------------|---------------------|-------------------|
| Market Claims | 80%+ | Industry data, analyst reports |
| Financial Claims | 85%+ | Model audit, comparable analysis |
| Technical Claims | 90%+ | Platform review, security audit |
| Legal Claims | 95%+ | Legal opinion, regulatory guidance |
| Partner Claims | 90%+ | Reference checks, background verification |

### Validation Status Badges

| Status | Meaning |
|--------|---------|
| VERIFIED | Fact-checked, cross-validated, sources confirmed |
| PARTIALLY VERIFIED | Some validation, needs additional confirmation |
| UNVERIFIED | No validation yet, requires research |
| FLAGGED | Conflicting sources or concerns identified |

### Deliverables
- [ ] Validation report with confidence scores
- [ ] Updated risk register
- [ ] Mitigation strategies documented
- [ ] Chief of Staff sign-off

### Prompt to Use
> "Please have the Chief of Staff conduct triple validation on all findings. Cross-reference claims, verify sources, and update the risk register with mitigations. Reference: MASTER_SME_TEAM_DOCUMENT.md - Part 1 (Chief of Staff Role)"

### SME Team Reference
Reference: MASTER_SME_TEAM_DOCUMENT.md - Part 1 (Chief of Staff Role) and Part 4 (Reference Requirements)

## Phase 4: Master Report and Strategic Planning

### Trigger
Triple validation complete, mitigations documented

### Tasks
1. Generate master report summarizing all findings
2. Identify remaining gaps and open questions
3. Recommend next steps and action owners
4. Create 8-week project plan with timeline
5. Define key performance indicators (KPIs)
6. Escalate critical decisions to stakeholder

### Master Report Structure

**Section 1: Executive Summary**
- Investment opportunity overview
- Key findings and validation status
- Critical risks and mitigations
- Recommended next steps

**Section 2: Market Analysis**
- Market sizing (TAM/SAM/SOM)
- Competitive landscape
- Customer segmentation
- Go-to-market strategy

**Section 3: Business Model**
- Revenue streams
- Unit economics
- Financial projections (bull/base/bear)
- Path to profitability

**Section 4: Operational Plan**
- Organizational structure
- Key roles and responsibilities
- Technology roadmap
- Execution timeline

**Section 5: Risk Analysis**
- Risk register with mitigations
- Contingency plans
- Governance structure
- Escalation protocols

**Section 6: Strategic Recommendations**
- Immediate actions (next 8 weeks)
- Medium-term priorities (months 3-6)
- Long-term vision (year 1+)
- Success metrics and KPIs

### Deliverables
- [ ] Master report document
- [ ] 8-week project plan
- [ ] KPI dashboard template
- [ ] Stakeholder decision points identified

### Prompt to Use
> "Please compile all findings into a Master Report following the structure in Phase 4. Include the 8-week project plan and KPI definitions. Have the Chief of Staff validate the final document. Reference: MASTER_SME_TEAM_DOCUMENT.md"

### SME Team Reference
Reference: MASTER_SME_TEAM_DOCUMENT.md - McKinsey Lens, Strategy SMEs

## Phase 5: McKinsey Business Plan Development

### Trigger
Master report approved, strategic direction confirmed

### Tasks
1. Develop comprehensive business plan
2. Create detailed operating model
3. Build organizational charts and role definitions
4. Develop financial model with multiple scenarios
5. Create competitive positioning framework
6. Document execution roadmap

### Business Plan Structure

**Part 1: Strategy and Market**
- Business overview and value proposition
- Market analysis and sizing
- Competitive positioning
- Strategic objectives

**Part 2: Operating Model**
- Organizational structure
- Key processes and workflows
- Technology architecture
- Governance framework

**Part 3: Financial Plan**
- Revenue model and projections
- Cost structure and unit economics
- Cash flow analysis
- Valuation and exit scenarios

**Part 4: Execution Plan**
- Roadmap and milestones
- Resource requirements
- Risk management
- Success metrics

**Part 5: Investment Case**
- Use of funds
- Expected returns
- Exit strategy
- Key assumptions

### Deliverables
- [ ] Comprehensive business plan document
- [ ] Financial model (Excel)
- [ ] Organizational chart
- [ ] Competitive positioning framework

### Prompt to Use
> "Please develop a McKinsey-style business plan based on the Master Report. Include all five parts with detailed analysis. Reference: MASTER_SME_TEAM_DOCUMENT.md - McKinsey and Company expert"

Reference: MASTER_SME_TEAM_DOCUMENT.md - McKinsey and Company, Financial SMEs

## Phase 6: Presentation and Funding Materials

### Trigger
Business plan complete and validated

### Tasks
1. Create investor presentation deck (20-30 slides)
2. Develop detailed presenter notes
3. Create executive summary (2 pages)
4. Build financial model for investor review
5. Prepare Q&A document
6. Create one-pager for initial outreach

### Presentation Structure

| Slide | Title | Content |
|-------|-------|---------|
| 1 | Cover | Company name, tagline, date |
| 2 | Problem | Market problem and opportunity |
| 3 | Solution | How the business solves the problem |
| 4 | Market Size | TAM/SAM/SOM with validation |
| 5 | Business Model | Revenue streams and unit economics |
| 6 | Competitive Advantage | Differentiation and defensibility |
| 7 | Go-to-Market | Customer acquisition strategy |
| 8 | Team | Organizational structure and key roles |
| 9 | Financial Projections | Revenue, EBITDA, cash flow |
| 10 | Use of Funds | Capital allocation |
| 11 | Milestones | Key execution milestones |
| 12 | Risk Mitigation | Key risks and strategies |
| 13 | Exit Strategy | Potential acquirers and multiples |
| 14 | Investment Highlights | Top reasons to invest |
| 15+ | Appendix | Supporting data and analysis |

### Deliverables
- [ ] Investor presentation deck
- [ ] Presenter notes
- [ ] Executive summary (2 pages)
- [ ] Q&A document
- [ ] One-pager

### Prompt to Use
> "Please create the investor presentation materials following the Phase 6 structure. Use the Storytelling Expert and Visual Design Expert for review. Reference: MASTER_SME_TEAM_DOCUMENT.md"

### SME Team Reference
Reference: MASTER_SME_TEAM_DOCUMENT.md - Storytelling Expert, Visual Design Expert, Investment SMEs

### Trigger

### Tasks

| Competitive Advantage | 45 seconds | Why this business wins |

## Phase 8: Project Execution and Monitoring

### Trigger
All materials complete, stakeholder engagement begins

### Tasks
1. Track project milestones against timeline
2. Monitor KPIs and success metrics
3. Update risk register monthly
4. Generate status reports
5. Escalate issues and blockers
6. Adjust plan based on market feedback

### Status Report Template

| Section | Content |
|---------|---------|
| Executive Summary | Overall status, key metrics, critical issues |
| Milestone Status | Completed, in-progress, upcoming |
| KPI Dashboard | Key metrics vs targets |
| Risk Register | Active risks, mitigations, new risks |
| Financial Status | Burn rate, runway, funding progress |
| Team Updates | Hiring, changes, capability gaps |
| Upcoming Priorities | Next 2-week focus areas |
| Escalations | Critical decisions needed |

### Deliverables
- [ ] Weekly status reports
- [ ] Monthly risk register updates
- [ ] KPI tracking dashboard
- [ ] Escalation log

### Prompt to Use
