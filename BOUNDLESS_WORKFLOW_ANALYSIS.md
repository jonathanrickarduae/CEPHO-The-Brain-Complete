# Boundless Workflow Template Analysis

## Overview
**Title:** Boundless Telecom — Meeting Preparation Workflow
**Purpose:** A repeatable process for preparing stakeholder meetings, generating briefing materials, and capturing intelligence.
**Prepared by:** Jonathan Rickard, Chief Strategy Officer

## Process Flow (6 Phases)

### Phase 1: Input Gathering
Collect all relevant source materials before analysis begins.

| Input Type | Description | Source |
|------------|-------------|--------|
| Business Plan Documents | Official plans, pitch decks, strategy documents | Founders / Stakeholders |
| Market Research | Sector analysis, competitor data, market sizing | Internal research / External sources |
| Financial Data | Projections, actuals, assumptions | Finance team / Founders |
| Stakeholder Information | Key parties, relationships, interests, history | Due diligence / Public records |

**Checklist:**
- [ ] All source documents collected and reviewed
- [ ] Key stakeholders identified with roles and interests
- [ ] Financial data validated or flagged for verification
- [ ] Market context understood

### Phase 2: Deep Dive Analysis
Produce a comprehensive analysis document covering all aspects of the opportunity.

| Section | Content |
|---------|---------|
| Sector Analysis | Market opportunity, trends, competitive landscape |
| Business Plan Review | Strategy, model, go-to-market, differentiation |
| Financial Model | Revenue projections, margins, break-even, sensitivities |
| Corporate Structure | Recommended entity structure, jurisdiction, tax considerations |
| Risk Assessment | Key risks, mitigations, contract protections required |
| Stakeholder Analysis | Key parties, interests, potential conflicts |

**Checklist:**
- [ ] Deep dive document drafted
- [ ] Financial model stress-tested
- [ ] Risks identified and mitigations proposed
- [ ] Corporate structure recommendations documented

### Phase 3: Briefing Materials
Create targeted briefing documents for specific meetings and audiences.

**3.1 Legal Briefing Paper** (For General Counsel or legal advisor meetings)

| Section | Purpose |
|---------|---------|
| Business Overview | 2-minute summary of the opportunity |
| Key Parties | Who's involved, their roles, interests |
| Corporate Structure | Recommended structure with rationale |
| Legal Concerns | Key risks with recommendations |
| Contract Framework | Supply agreement and equity protections |
| Key Questions | What needs to be investigated, by whom |

**3.2 Meeting Discussion Points** (One-page talking points for stakeholder meetings)

| Section | Purpose |
|---------|---------|
| Objective | What the meeting should achieve |
| Discussion Topics | Numbered list of areas to explore |
| Key Questions | What we need to understand |
| Approach | How to position the conversation |
| Key Advice | Strategic guidance for the meeting |

### Phase 4: Audio Briefing
Generate a spoken briefing for stakeholders to listen to before meetings.

| Element | Description |
|---------|-------------|
| Introduction | Who the briefing is from, purpose, context |
| Document Summary | What's attached and what's in each document |
| Meeting Agenda | What needs to be achieved |
| Key Areas | Walk through the discussion points |
| Strategic Advice | How to approach the meeting |
| Close | Next steps, offer of support |

**Voice:** British female (Victoria Sterling), professional, slightly posh, soft southern accent.
**Tool:** ElevenLabs with Charlotte or Dorothy voice.

### Phase 5: Meeting Pack Delivery
Combine all materials into a single meeting pack.

| Deliverable | Format | Purpose |
|-------------|--------|---------|
| Deep Dive Report | PDF | Background and reference |
| Briefing Paper | PDF | Targeted summary for the meeting |
| Discussion Points | PDF (1-page) | In-meeting reference |
| Audio Briefing | WAV/MP3 | Listen before the meeting |

### Phase 6: Post-Meeting Capture
After the meeting, capture intelligence and update the analysis.

| Capture | Action |
|---------|--------|
| Intelligence Gathered | Update deep dive with new information |
| Action Items | Log and assign owners |
| Next Steps | Schedule follow-up, update briefing materials |
| Risk Updates | Revise risk register if new concerns identified |

**Checklist:**
- [ ] Meeting notes captured
- [ ] Intelligence fed back into deep dive document
- [ ] Action items logged with owners and deadlines
- [ ] Next meeting scheduled and briefing cycle restarted
