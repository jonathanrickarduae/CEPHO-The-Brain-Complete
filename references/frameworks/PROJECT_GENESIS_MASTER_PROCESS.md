Upload this document alongside **MASTER_SME_TEAM_DOCUMENT.md** to any new Manus task. This provides the complete process framework for transforming raw business concepts into investor-ready materials.

## Phase 0: Voice Note Intake and Scope Alignment

### Trigger
Stakeholder initiates project with voice note, document, or description

### Tasks
1. Transcribe voice note (if applicable)
2. Extract key elements:
   - Opportunity description
   - Key stakeholders
   - Timeline expectations
   - Budget parameters
   - Success objectives
3. Pre-populate project intake form
4. Generate scope alignment summary
5. Confirm scope with stakeholder
6. **Initialize 100% Optimization Scorecard** (NEW)
   - Create project scorecard with all 50 categories
   - Set initial baseline scores (0 for unknown, estimated for known)
   - Identify which domains are most critical for this project type
   - Generate initial heat map showing gaps

### Deliverables
- [ ] Transcribed voice note
- [ ] Scope alignment confirmation
- [ ] Project intake form completed
- [ ] **100% Optimization Scorecard initialized with baseline scores**
- [ ] **Initial heat map showing gap areas**

### Prompt to Use
> "Please extract the key elements from this opportunity description and create a scope alignment summary. Identify: opportunity, stakeholders, timeline, budget, and objectives."

Reference: MASTER_SME_TEAM_DOCUMENT.md - No experts needed at this phase

## Phase 3: Triple Validation and Risk Mitigation

### Trigger
Deep dive research complete

### Tasks
1. Validate all claims through primary SME expert
2. Cross-validate through secondary perspective
3. Triple-check through tertiary perspective
4. **Customer Focus Group Validation** (NEW)
   - Select relevant customer personas from the 100+ persona database
   - Deploy targeted survey on product/service concept
   - Collect feedback on value proposition and pricing
   - Aggregate customer viewpoints for decision support
5. Flag any unverified claims
6. Generate validation report with confidence scores
7. Update risk register with research findings
8. Document mitigations for each risk
9. **Check Prior Research** (NEW)
   - Query Insights Repository for existing relevant research
   - Reference previous customer feedback on similar concepts
   - Avoid duplicate research where insights already exist

### Validation Framework

| Claim Category | Confidence Threshold | Validation Method |
|----------------|---------------------|-------------------|
| Market Claims | 80%+ | Industry data, analyst reports |
| Financial Claims | 85%+ | Model audit, comparable analysis |
| Technical Claims | 90%+ | Platform review, security audit |
| Legal Claims | 95%+ | Legal opinion, regulatory guidance |
| Partner Claims | 90%+ | Reference checks, background verification |
| **Customer Claims** | 75%+ | Customer Focus Group survey, persona feedback |

### Validation Status Badges

| Status | Meaning |
|--------|---------|
| VERIFIED | Fact-checked, cross-validated, sources confirmed |
| PARTIALLY VERIFIED | Some validation, needs additional confirmation |
| UNVERIFIED | No validation yet, requires research |
| FLAGGED | Conflicting sources or concerns identified |

### Customer Focus Group Integration

| Step | Action | Output |
|------|--------|--------|
| 1 | Select personas by industry, region, income | Target persona list |
| 2 | Design survey (max 10 questions) | Survey template |
| 3 | Deploy to selected personas | Response collection |
| 4 | Aggregate feedback | Customer validation report |
| 5 | Store insights | Insights Repository entry |

### Deliverables
- [ ] Validation report with confidence scores
- [ ] Updated risk register
- [ ] Mitigation strategies documented
- [ ] **Customer Focus Group feedback report** (NEW)
- [ ] **Insights stored in repository** (NEW)
- [ ] **100% Optimization Scorecard updated with validation scores**
- [ ] Chief of Staff sign-off

### Prompt to Use
> "Please have the Chief of Staff conduct triple validation on all findings, including Customer Focus Group feedback. Cross-reference claims, verify sources, deploy customer survey, and update the risk register with mitigations. Reference: MASTER_SME_TEAM_DOCUMENT.md - Part 1 (Chief of Staff Role)"

## Phase 4: Master Report and Strategic Planning

### Trigger
Triple validation complete, mitigations documented

### Tasks
1. Generate master report summarizing all findings
2. Identify remaining gaps and open questions
3. Recommend next steps and action owners
4. Create 8-week project plan with timeline
5. Define key performance indicators (KPIs)
6. Escalate critical decisions to stakeholder
7. **Individual SME Scoring Assessment** (NEW)
   - Each SME expert scores relevant KPI categories (0-100)
   - Capture rationale for each score
   - Detect outliers (significant deviations from panel average)
   - Flag discrepancies for Chief of Staff investigation
8. **Generate KPI Heat Map** (NEW)
   - Visualize all 50 KPI categories with color coding
   - Green (90+) to Red (0-39) based on scores
   - Enable multi-perspective views (CoS, SME, Customer)
   - Identify weak spots requiring attention

### Master Report Structure

**Section 1: Executive Summary**
- Investment opportunity overview
- Key findings and validation status
- Critical risks and mitigations
- Recommended next steps

**Section 2: Market Analysis**
- Market sizing (TAM/SAM/SOM)
- Competitive landscape
- Customer segmentation
- Go-to-market strategy

**Section 3: Business Model**
- Revenue streams
- Unit economics
- Financial projections (bull/base/bear)
- Path to profitability

**Section 4: Operational Plan**
- Organizational structure
- Key roles and responsibilities
- Technology roadmap
- Execution timeline

**Section 5: Risk Analysis**
- Risk register with mitigations
- Contingency plans
- Governance structure
- Escalation protocols

**Section 6: Strategic Recommendations**
- Immediate actions (next 8 weeks)
- Medium-term priorities (months 3-6)
- Long-term vision (year 1+)
- Success metrics and KPIs

### 50 Category KPI Framework

| Domain | Categories | Key Focus |
|--------|------------|----------|
| Strategy | 6 | Vision, positioning, business model |
| Technology | 6 | Architecture, security, scalability |
| Product | 5 | Quality, innovation, roadmap |
| Customer | 5 | Satisfaction, retention, support |
| Operations | 5 | Efficiency, quality, supply chain |
| Finance | 5 | Revenue, profitability, cash flow |
| People | 5 | Talent, culture, leadership |
| Governance | 5 | Compliance, risk, ethics |
| Innovation | 4 | R&D, IP, partnerships |
| Market | 4 | Position, growth, brand |

### Deliverables
- [ ] Master report document
- [ ] 8-week project plan
- [ ] KPI dashboard template
- [ ] Stakeholder decision points identified
- [ ] **Individual SME scoring completed** (NEW)
- [ ] **KPI Heat Map generated** (NEW)
- [ ] **Outlier analysis report** (NEW)

### Prompt to Use

Have the Chief of Staff validate the final document and investigate any scoring outliers. Reference: MASTER_SME_TEAM_DOCUMENT.md"

## Phase 5: McKinsey Business Plan Development

### Trigger
Master report approved, strategic direction confirmed

### Tasks
1. Develop comprehensive business plan
2. Create detailed operating model
3. Build organizational charts and role definitions
4. Develop financial model with multiple scenarios
5. Create competitive positioning framework
6. Document execution roadmap

### Business Plan Structure

**Part 1: Strategy and Market**
- Business overview and value proposition
- Market analysis and sizing
- Competitive positioning
- Strategic objectives

**Part 2: Operating Model**
- Organizational structure
- Key processes and workflows
- Technology architecture
- Governance framework

**Part 3: Financial Plan**
- Revenue model and projections
- Cost structure and unit economics
- Cash flow analysis
- Valuation and exit scenarios

**Part 4: Execution Plan**
- Roadmap and milestones
- Resource requirements
- Risk management
- Success metrics

**Part 5: Investment Case**
- Use of funds
- Expected returns
- Exit strategy
- Key assumptions

### Deliverables
- [ ] Comprehensive business plan document

- [ ] Organizational chart
- [ ] Competitive positioning framework
- [ ] **100% Optimization Scorecard updated with business plan scores**

## Phase 6: Presentation and Funding Materials

### Trigger
Business plan complete and validated

### Tasks
1. Create investor presentation deck (20-30 slides)
2. Develop detailed presenter notes
3. Create executive summary (2 pages)
4. Build financial model for investor review
5. Prepare Q&A document
6. Create one-pager for initial outreach

### Presentation Structure

| Slide | Title | Content |
|-------|-------|---------|
| 1 | Cover | Company name, tagline, date |
| 2 | Problem | Market problem and opportunity |
| 3 | Solution | How the business solves the problem |
| 4 | Market Size | TAM/SAM/SOM with validation |
| 5 | Business Model | Revenue streams and unit economics |
| 6 | Competitive Advantage | Differentiation and defensibility |
| 7 | Go-to-Market | Customer acquisition strategy |
| 8 | Team | Organizational structure and key roles |
| 9 | Financial Projections | Revenue, EBITDA, cash flow |
| 10 | Use of Funds | Capital allocation |
| 11 | Milestones | Key execution milestones |
| 12 | Risk Mitigation | Key risks and strategies |
| 13 | Exit Strategy | Potential acquirers and multiples |
| 14 | Investment Highlights | Top reasons to invest |
| 15+ | Appendix | Supporting data and analysis |

- [ ] Investor presentation deck
