# Developer Onboarding & 30-Day Plan

This document outlines the structured onboarding plan for all new engineers joining the CEPHO.AI team.

## Week 1: Setup & Immersion

- **Day 1:** Get access to all systems (GitHub, Slack, Render, Supabase). Set up local development environment by following the `README.md`.
- **Day 2:** Pair with your onboarding buddy to walk through the codebase and architecture.
- **Day 3:** Pick up your first "good first issue" — a small, well-defined bug fix or improvement.
- **Day 4:** Submit your first Pull Request (PR) and go through the code review process.
- **Day 5:** Merge your first PR to the `develop` branch.

## Week 2: First Feature

- **Goal:** Ship a small, self-contained feature from start to finish.
- **Process:** Pick a small feature from the backlog. Write the code, tests, and documentation. Get it reviewed and merged.
- **Shadowing:** Shadow the on-call engineer for one day to understand the incident response process.

## Week 3: Deeper Dive

- **Goal:** Gain a deeper understanding of a specific part of the system.
- **Process:** Work on a more complex feature or bug that requires a deeper dive into a specific service or component.
- **Documentation:** Contribute to the documentation for the area you are working on.

## Week 4: On-Call & Ownership

- **Goal:** Become a fully contributing member of the team.
- **Process:** Join the on-call rotation as the secondary on-call engineer.
- **Ownership:** Take ownership of a small feature area, becoming the go-to person for that part of the system.
