# CEPHO.Ai Master Design Guidelines

**Version:** 1.0  
**Effective Date:** 17 January 2026  
**Status:** MANDATORY for all outputs  
**Owner:** Chief of Staff

---

## 1. Purpose and Scope

This document serves as the **single source of truth** for all CEPHO.Ai branded outputs. Every document, report, presentation, briefing paper, or any other deliverable MUST comply with these guidelines before delivery to the user.

**Compliance is mandatory.** No document may be delivered without passing Chief of Staff validation against these specifications.

---

## 2. Brand Identity

### 2.1 Brand Name

| Element | Specification |
|---------|---------------|
| Full Name | CEPHO.Ai |
| Capitalization | CEPHO in capitals, .Ai with capital A, lowercase i |
| Never Use | Cepho, cepho, CEPHO.AI, Cepho.ai |

### 2.2 Brand Positioning

CEPHO.Ai represents a premium, professional AI platform. All outputs must reflect:

- **Authority:** Expert level quality and accuracy
- **Clarity:** Clean, uncluttered, easy to read
- **Professionalism:** Consulting firm standard
- **Sophistication:** Modern but not trendy

## 3. Color Palette

### 3.1 Primary Colors

| Color Name | Hex Code | RGB | Usage |
|------------|----------|-----|-------|
| CEPHO Black | #000000 | 0, 0, 0 | Primary text, headers, logo background |
| CEPHO White | #FFFFFF | 255, 255, 255 | Page backgrounds, reversed text on black |
| CEPHO Dark Grey | #333333 | 51, 51, 51 | Secondary text, subheadings |
| CEPHO Mid Grey | #666666 | 102, 102, 102 | Tertiary text, captions, metadata |
| CEPHO Light Grey | #F3F4F6 | 243, 244, 246 | Background fills, table headers, dividers |

### 3.2 Accent Color

| Color Name | Hex Code | RGB | Usage |
|------------|----------|-----|-------|
| CEPHO Pink | #FF006E | 255, 0, 110 | Accent bars, highlights, key metrics, calls to action |

### 3.3 Score/Status Colors

| Score Range | Color Name | Hex Code | Text Color |
|-------------|------------|----------|------------|
| 80-100 | Excellent Green | #22C55E | White (#FFFFFF) |
| 70-79 | Good Lime | #84CC16 | Black (#000000) |
| 60-69 | Satisfactory Yellow | #EAB308 | Black (#000000) |
| 40-59 | Attention Orange | #F97316 | White (#FFFFFF) |
| 30-39 | Critical Red | #EF4444 | White (#FFFFFF) |
| 0-29 | Severe Dark Red | #DC2626 | White (#FFFFFF) |

### 3.4 Color Usage Rules

1. **Primary documents:** Black, white, and grey only (no accent colors in body text)
2. **Accent usage:** Pink (#FF006E) for left-edge bars, key metrics, and important callouts only
3. **Score cells:** Always use white text on dark backgrounds (green 80+, orange, red)
4. **Score cells:** Always use black text on light backgrounds (lime, yellow)
5. **Never use:** Blue, purple, teal, or any color not specified above

## 4. Typography

### 4.1 Font Family

| Priority | Font | Fallback |
|----------|------|----------|
| Primary | Calibri | Arial, Helvetica, sans-serif |

### 4.2 Font Sizes and Weights

| Element | Size | Weight | Color |
|---------|------|--------|-------|
| Document Title | 24pt | Bold | Black (#000000) |
| Section Header (H1) | 18pt | Bold | Black (#000000) |
| Subsection Header (H2) | 14pt | Bold | Black (#000000) |
| Body Text | 11pt | Regular | Dark Grey (#333333) |
| Table Header | 10pt | Bold | Mid Grey (#666666) |
| Table Body | 9pt | Regular | Black (#000000) |
| Caption/Metadata | 9pt | Regular | Mid Grey (#666666) |
| Footer | 8pt | Regular | Mid Grey (#666666) |
| Page Numbers | DO NOT INCLUDE | - | - |

### 4.3 Typography Rules

1. **No page numbers** on any document
2. **No hyphens** in compound words (to avoid AI appearance)
3. **Line height:** 1.4 for body text, 1.2 for headers
4. **Paragraph spacing:** 10pt after paragraphs
5. **Never use:** Italic for emphasis (use bold sparingly instead)

## 6. Document Types and Templates

### 6.1 Report (KPI, Assessment, Research)

| Element | Specification |
|---------|---------------|
| Title Page | Document title centered, 24pt bold, stage/framework subtitle 12pt grey |
| Executive Summary Box | Light grey background with pink left accent bar (3mm) |
| Data Tables | Light grey header row, alternating white rows, 1pt borders |
| Score Cells | Colored background per Section 3.3, appropriate text contrast |
| Validation Section | Approval status with date, framework reference box |

### 6.2 Briefing Paper

| Element | Specification |
|---------|---------------|
| Length | Maximum 2 pages |
| Structure | Context, Key Points, Recommendations, Next Steps |
| Format | Paragraphs, not bullet points |
| Callout Boxes | Light grey with pink left accent for key insights |

### 6.3 Executive Summary

| Element | Specification |
|---------|---------------|
| Length | Maximum 1 page |
| Structure | Situation, Analysis, Recommendation |
| Key Metrics | Highlighted in pink accent boxes |
| No specific numbers | Use qualitative statements unless critical |

### 6.4 Presentation/Pitch Deck

| Element | Specification |
|---------|---------------|
| Aspect Ratio | 16:9 |
| Slide Background | White |
| Title Slides | Black text, pink accent underline |
| Content Slides | Maximum 6 points per slide |
| Images | Real photographs, not stock graphics |

### 6.5 Meeting Notes

| Element | Specification |
|---------|---------------|
| Header | Meeting title, date, attendees |
| Structure | Agenda, Discussion Points, Decisions, Actions |
| Action Items | Table format with owner, due date, status |

## 9. Quality Checklist

Before any document is delivered, the Chief of Staff MUST verify:

### 9.1 Brand Compliance

- [ ] CEPHO.Ai spelled correctly throughout
- [ ] Logo block present and correctly positioned
- [ ] Pink accent line below header
- [ ] Footer text present and correct
- [ ] No page numbers

### 9.2 Color Compliance

- [ ] Only approved colors used (black, white, grey, pink accent)
- [ ] Score cells have correct background colors
- [ ] Text contrast is readable (white on dark, black on light)
- [ ] Pink accent used sparingly and appropriately

### 9.3 Typography Compliance

- [ ] Calibri font used throughout
- [ ] Correct font sizes per element type
- [ ] No hyphens in compound words
- [ ] No italic text
- [ ] Proper line spacing

### 9.4 Structure Compliance

- [ ] Correct margins applied
- [ ] Section dividers present
- [ ] Tables formatted correctly
- [ ] Callout boxes styled correctly

### 9.5 Content Quality

- [ ] Professional tone throughout
- [ ] No AI self-references
- [ ] Factual accuracy verified
- [ ] Complete and coherent

### 9.6 File Compliance

- [ ] Correct naming convention
- [ ] Author field blank or "X"
- [ ] Appropriate file format

## 10. Validation Process

### 10.1 Mandatory Gate

**No document may be delivered to the user without passing Chief of Staff validation.**

### 10.2 Validation Flow

```
1. Document Generated
2. Chief of Staff Review (this checklist)
3. IF PASS → Deliver to User
4. IF FAIL → Regenerate (user never sees failures)
```

### 10.3 Failure Handling

If a document fails validation:

1. Identify specific failures from checklist
2. Regenerate document with corrections
3. Re-validate
4. Repeat until all checks pass
5. Only then deliver to user
