# Chief of Staff Sprint Process

## Overview

The Chief of Staff Sprint is a structured approach for managing large development tasks and initiatives. It provides a clear framework for tracking progress throughout the day, ensuring accountability, and maintaining momentum on complex projects.

## Decision Making Approach

During sprint execution, use **Digital Twin Thinking** and **AI SME Expert Judgment** for all implementation decisions:

1. **Digital Twin Thinking**: Apply knowledge of the user's preferences, past decisions, and established patterns to make choices that align with their vision
2. **AI SME Expert Judgment**: For UI/UX decisions, apply expert design principles. For technical decisions, apply engineering best practices. For business decisions, apply strategic frameworks
3. **No Stopping**: Do not pause to ask questions during execution. Make the best decision based on available context and expert knowledge
4. **Log Issues**: If a decision cannot be made or an action fails, log it as a comment and move to the next item
5. **Batch Review**: Issues are reviewed in bulk at the end of the sprint, not during execution

## When to Use

Use this process when:
- A task has more than 10 individual action items
- Multiple areas or domains need coordinated work
- Progress needs to be tracked and reported throughout the day
- Items may need to be deferred or commented on during execution

## Process Steps

### 1. Sprint Planning (Start of Day)

1. **Gather all action items** from various sources:
   - SME Panel recommendations
   - UI/UX review findings
   - Development pathway items
   - User-suggested features
   - Bug fixes and improvements

2. **Consolidate into single Action List**:
   - Assign sequential numbers (1, 2, 3...) for easy reference
   - Group by logical sections
   - Include columns: #, Action, Status, Comments

3. **Prioritize sections**:
   - HIGH priority sections first
   - MEDIUM priority sections follow
   - Tag items that are blockers or dependencies

4. **Generate Sprint Document**:
   - Create PDF following CEPHO design guidelines
   - Include Quick Reference table with section ranges
   - Ensure all text is black (#000000) - no light grey

### 2. Sprint Execution (Throughout Day)

1. **Work through items sequentially** within each section

2. **Update status** as items complete:
   - ☐ → ☑ when complete
   - Add comments for any issues or blockers

3. **Communication shortcuts**:
   - "Complete action 5" - marks item 5 as done
   - "Defer action 23" - moves to later/tomorrow
   - "Issue on action 12: [description]" - logs blocker

4. **Regular check-ins**:
   - Morning: Review priorities
   - Midday: Progress update
   - End of day: Summary and carryover

### 3. Sprint Review (End of Day)

1. **Calculate completion rate**:
   - Total items completed / Total items planned
   - Target: 70%+ completion rate

2. **Document blockers and issues**:
   - What couldn't be completed and why
   - Dependencies identified
   - Resources needed

3. **Carryover items**:
   - Uncompleted items roll to next sprint
   - Re-prioritize based on learnings

4. **Update KPI Dashboard**:
   - Log sprint metrics
   - Track velocity over time

## Document Format

### Action List Template

| # | Action | Status | Comments |
|---|--------|--------|----------|
| 1 | [Action description] | ☐ | |
| 2 | [Action description] | ☐ | |

### Quick Reference Table

| Section | Items | Range | Priority |
|---------|-------|-------|----------|
| 1. [Section Name] | [count] | [start-end] | HIGH/MEDIUM |

### Design Requirements

- **Font**: Inter or Calibri
- **Text color**: Black (#000000) only - NO light grey
- **Accent color**: Pink (#E91E8C) for headers and highlights
- **Header style**: Black background with white text
- **Section headers**: Left pink border with grey background

## Integration with CEPHO Systems

### Chief of Staff Dashboard
- Sprint progress displayed on main dashboard
- Percentage completion shown in real-time
- Blockers flagged for attention

### KPI Reporting
- Sprint velocity tracked as KPI metric
- Completion rates feed into overall score
- SME recommendations tracked through sprints

### Template Repository
- Action List template stored in repository
- Consistent formatting across all sprints
- Version controlled for improvements

## Example Sprint Flow

```
08:00 - Sprint Planning
       - Consolidate 140 items from various sources
       - Generate Action List PDF
       - Review priorities with stakeholder

09:00 - Sprint Execution Begins
       - Start with Section 1 (HIGH priority)
       - "Complete action 1" → UI design language applied
       - "Complete action 2" → Logo updated

12:00 - Midday Check-in
       - 35 items complete (25%)
       - 2 blockers identified
       - Adjust afternoon priorities

17:00 - Sprint Review
       - 98 items complete (70%)
       - 42 items deferred to tomorrow
       - Document learnings
       - Update KPI dashboard
```

## Process Owner

**Chief of Staff** - Responsible for:
- Creating sprint documents
- Tracking progress
- Reporting status
- Managing carryover items
